Poglavlje 0: Uvod u ASP.NET MVC

U ovom poglavlju upoznat ćete se s osnovnim konceptima ASP.NET MVC obrasca, strukturom projekta i ključnim datotekama koje čine temelj svake MVC aplikacije.

Šta je MVC (Model-View-Controller)?

MVC je arhitekturalni obrazac koji dijeli aplikaciju u tri glavne komponente:

Model

Predstavlja podatke i poslovnu logiku. U našem projektu, modeli dolaze iz BHAS.DbFirst projekta (Entity Framework Database First pristup).

Primjeri: Department, Employee, Project, Region

View

Odgovoran je za prikaz podataka korisniku. Koristi Razor sintaksu (.cshtml datoteke) za kombinovanje HTML-a i C# koda.

Primjeri: Index.cshtml, Create.cshtml, Edit.cshtml

Controller

Prima korisničke zahtjeve (HTTP GET/POST), obrađuje ih, komunicira s modelom i odabire odgovarajući View za odgovor.

Primjeri: HomeController, DepartmentController

Kako radi tok zahtjeva u MVC-u:
1. Korisnik otvara URL npr. /Department/Details/5
2. Routing sistem pronalazi DepartmentController i poziva akciju Details s parametrom id = 5
3. Controller dohvaća podatke iz baze podataka (Model)
4. Controller prosljeđuje podatke View-u
5. View renderira HTML koji se šalje korisniku

Ključne datoteke

1. Global.asax.cs — Ulazna tačka aplikacije

Ova datoteka se izvršava kada se aplikacija pokrene. Registruje sve potrebne konfiguracije:

using System.Web.Mvc;
using System.Web.Optimization;
using System.Web.Routing;

namespace AS.MVCDemo
{
    public class MvcApplication : System.Web.HttpApplication
    {
        protected void Application_Start()
        {
            AreaRegistration.RegisterAllAreas();
            FilterConfig.RegisterGlobalFilters(GlobalFilters.Filters);
            RouteConfig.RegisterRoutes(RouteTable.Routes);
            BundleConfig.RegisterBundles(BundleTable.Bundles);
        }
    }
}

Metoda Application_Start() se poziva jednom kada aplikacija startuje i registruje:

  • Areas — Regije za organizaciju velikih projekata
  • Filters — Globalni filteri (npr. za obradu grešaka)
  • Routes — URL routing pravila
  • Bundles — Grupe CSS i JavaScript datoteka

2. RouteConfig.cs — Kako MVC razumije URL-ove

Routing je mehanizam koji mapira URL adrese na controller akcije. Evo defaultne konfiguracije:

using System.Web.Mvc;
using System.Web.Routing;

namespace AS.MVCDemo
{
    public class RouteConfig
    {
        public static void RegisterRoutes(RouteCollection routes)
        {
            routes.IgnoreRoute("{resource}.axd/{*pathInfo}");

            routes.MapRoute(
                name: "Default",
                url: "{controller}/{action}/{id}",
                defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
            );
        }
    }
}
Kako routing radi:

URL obrazac je {controller}/{action}/{id} gdje je:
  • {controller} — ime kontrolera (bez "Controller" sufiksa)
  • {action} — ime akcije (metode u kontroleru)
  • {id} — opcioni parametar

URL Controller Action id
/ HomeController Index() -
/Department DepartmentController Index() -
/Department/Details/5 DepartmentController Details(5) 5
/Employee/Edit/3 EmployeeController Edit(3) 3
/Employee/Create EmployeeController Create() -

3. BundleConfig.cs — Grupisanje CSS i JS datoteka

Bundle-ovi grupišu više CSS ili JavaScript datoteka u jednu referencu, što poboljšava performanse:

using System.Web.Optimization;

namespace AS.MVCDemo
{
    public class BundleConfig
    {
        public static void RegisterBundles(BundleCollection bundles)
        {
            bundles.Add(new ScriptBundle("~/bundles/jquery").Include(
                        "~/Scripts/jquery-{version}.js"));

            bundles.Add(new ScriptBundle("~/bundles/jqueryval").Include(
                        "~/Scripts/jquery.validate*"));

            bundles.Add(new ScriptBundle("~/bundles/modernizr").Include(
                        "~/Scripts/modernizr-*"));

            bundles.Add(new Bundle("~/bundles/bootstrap").Include(
                      "~/Scripts/bootstrap.js"));

            bundles.Add(new StyleBundle("~/Content/css").Include(
                      "~/Content/bootstrap.css",
                      "~/Content/site.css"));
        }
    }
}

U View-ovima koristimo ove bundlove ovako:

  • @Styles.Render("~/Content/css") — učitava bootstrap.css i site.css
  • @Scripts.Render("~/bundles/jquery") — učitava jQuery
  • @Scripts.Render("~/bundles/jqueryval") — učitava validacijske skripte

DbContext — StateStatisticsDBEntities

Svi kontroleri koriste StateStatisticsDBEntities klasu za pristup bazi podataka. Ova klasa nasljeđuje DbContext iz Entity Framework-a i izlaže sljedeće DbSet kolekcije:

public virtual DbSet<Department> Departments { get; set; }
public virtual DbSet<Employee> Employees { get; set; }
public virtual DbSet<Project> Projects { get; set; }
public virtual DbSet<Region> Regions { get; set; }
public virtual DbSet<EconomicData> EconomicDatas { get; set; }
public virtual DbSet<Report> Reports { get; set; }
public virtual DbSet<ReportData> ReportDatas { get; set; }

U kontrolerima ćemo ga koristiti ovako:

using (var db = new StateStatisticsDBEntities())
{
    var departments = db.Departments.ToList();
    return View(departments);
}
using statement: using osigurava da se DbContext pravilno zatvori i oslobode resursi nakon što završimo s upitom. Ovo je važno za sprečavanje curenja memorije (memory leaks).
