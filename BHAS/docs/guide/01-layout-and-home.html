Poglavlje 1: Layout stranica i HomeController

U ovom poglavlju kreirat ćemo zajedničku layout stranicu koja će se koristiti na svim stranicama, te jednostavan HomeController s početnom stranicom.

Šta je Layout stranica?

Layout stranica je master template koji definira zajedničku strukturu svih stranica: navigaciju, zaglavlje, podnožje, te reference na CSS i JavaScript datoteke. Umjesto da svaka stranica ponavlja isti HTML, sve stranice dijele jedan layout.

Ključni elementi Layout stranice:
  • @RenderBody() — Označava mjesto gdje će se ubaciti sadržaj svake pojedinačne stranice
  • @RenderSection("scripts", required: false) — Opciona sekcija za dodatne skripte specifične za stranicu
  • @Styles.Render() i @Scripts.Render() — Učitavaju CSS i JS bundlove definirane u BundleConfig.cs
  • @Html.ActionLink() — Generira HTML link koji vodi na određenu controller akciju

Korak 1: Kreiranje _Layout.cshtml

Lokacija datoteke: Views/Shared/_Layout.cshtml
Desni klik na folder Views/Shared → Add → New Item → MVC 5 Layout Page → ime: _Layout.cshtml

Layout stranica je HTML dokument koji koristi Razor sintaksu. Otkucajte sljedeći kod:

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>@ViewBag.Title - AS.MVCDemo</title>
    @Styles.Render("~/Content/css")
    @Scripts.Render("~/bundles/modernizr")
</head>
<body>
    <nav class="navbar navbar-expand-sm navbar-dark bg-dark">
        <div class="container">
            @Html.ActionLink("AS.MVCDemo", "Index", "Home", new { area = "" }, new { @class = "navbar-brand" })
            <button type="button" class="navbar-toggler" data-bs-toggle="collapse" data-bs-target=".navbar-collapse"
                    title="Toggle navigation" aria-controls="navbarSupportedContent"
                    aria-expanded="false" aria-label="Toggle navigation">
                <span class="navbar-toggler-icon"></span>
            </button>
            <div class="collapse navbar-collapse d-sm-inline-flex justify-content-between">
                <ul class="navbar-nav flex-grow-1">
                    <li>@Html.ActionLink("Početna", "Index", "Home", new { area = "" }, new { @class = "nav-link" })</li>
                    <li>@Html.ActionLink("Odjeli", "Index", "Department", new { area = "" }, new { @class = "nav-link" })</li>
                    <li>@Html.ActionLink("Zaposlenici", "Index", "Employee", new { area = "" }, new { @class = "nav-link" })</li>
                    <li>@Html.ActionLink("Projekti", "Index", "Project", new { area = "" }, new { @class = "nav-link" })</li>
                    <li>@Html.ActionLink("Regije", "Index", "Region", new { area = "" }, new { @class = "nav-link" })</li>
                    <li>@Html.ActionLink("Ekon. podaci", "Index", "EconomicData", new { area = "" }, new { @class = "nav-link" })</li>
                    <li>@Html.ActionLink("Izvještaji", "Index", "Report", new { area = "" }, new { @class = "nav-link" })</li>
                </ul>
            </div>
        </div>
    </nav>
    <div class="container body-content" style="margin-top: 20px;">
        @RenderBody()
        <hr />
        <footer>
            <p>&copy; @DateTime.Now.Year - AS.MVCDemo</p>
        </footer>
    </div>

    @Scripts.Render("~/bundles/jquery")
    @Scripts.Render("~/bundles/bootstrap")
    @RenderSection("scripts", required: false)
</body>
</html>

Objašnjenje koda

Linija kodaObjašnjenje
@ViewBag.Title Dinamički naslov stranice. Svaka stranica postavlja ovu vrijednost u svom View-u.
@Styles.Render("~/Content/css") Učitava CSS bundle koji smo definirali u BundleConfig.cs (bootstrap.css + site.css).
@Html.ActionLink("Početna", "Index", "Home", ...) Generira <a> tag koji vodi na HomeController / Index akciju. Parametri su: tekst linka, akcija, kontroler, route values, HTML atributi.
@RenderBody() Ovdje se ubacuje sadržaj svake stranice. Ovo je najvažniji dio Layout stranice.
@DateTime.Now.Year Ispisuje trenutnu godinu u footer-u.
@RenderSection("scripts", required: false) Opciona sekcija. Stranice mogu definirati @section scripts { ... } za dodatne JS skripte.
Html.ActionLink objašnjenje:
@Html.ActionLink("Tekst", "Akcija", "Kontroler", routeValues, htmlAttributes)

Primjer: @Html.ActionLink("Odjeli", "Index", "Department", new { area = "" }, new { @class = "nav-link" })
Ovo generira: <a class="nav-link" href="/Department">Odjeli</a>

new { area = "" } osigurava da link radi ispravno čak i ako koristimo Areas.

Korak 2: Kreiranje _ViewStart.cshtml

Lokacija datoteke: Views/_ViewStart.cshtml
Ova datoteka se automatski izvršava prije svakog View-a i postavlja default Layout. 
@{
    Layout = "~/Views/Shared/_Layout.cshtml";
}

Zahvaljujući ovoj datoteci, ne moramo u svakom View-u ručno postavljati Layout. Sve stranice automatski koriste _Layout.cshtml.

Kako _ViewStart radi: Kada MVC treba prikazati bilo koji View, prvo provjerava da li postoji _ViewStart.cshtml u istom folderu ili u roditeljskom folderu. Ako postoji, izvršava ga prije View-a. Zato naš _ViewStart.cshtml stoji u root Views/ folderu — primjenjuje se na sve View-ove.

Korak 3: Kreiranje HomeController

Lokacija datoteke: Controllers/HomeController.cs
Desni klik na folder Controllers → Add → Controller → MVC 5 Controller - Empty → ime: HomeController

HomeController je najjednostavniji kontroler. Ima samo jednu akciju — Index() — koja prikazuje početnu stranicu:

using System.Web.Mvc;

namespace AS.MVCDemo.Controllers
{
    public class HomeController : Controller
    {
        public ActionResult Index()
        {
            ViewBag.Title = "Početna";
            return View();
        }
    }
}

Objašnjenje koda

ElementObjašnjenje
using System.Web.Mvc Namespace koji sadrži baznu klasu Controller i sve MVC helpere.
: Controller Svaki kontroler mora naslijediti klasu Controller. To mu daje pristup metodama kao što su View(), RedirectToAction(), itd.
ActionResult Povratni tip akcije. ActionResult je bazni tip koji može biti View, Redirect, JSON, File, itd.
ViewBag.Title ViewBag je dinamički objekt za prosljeđivanje podataka iz kontrolera u View. Ovdje postavljamo naslov stranice.
return View() Vraća View koji se podrazumijevano traži u Views/Home/Index.cshtml (po konvenciji: naziv kontrolera / naziv akcije).
ViewBag vs ViewData vs Model:
  • ViewBag — Dinamički objekt (ne zahtijeva cast). Koristimo za male podatke poput naslova.
  • ViewData — Dictionary (ViewData["Title"]). Zahtijeva cast kod čitanja.
  • Model — Strongly-typed. Najbolji način za prosljeđivanje podataka. Koristit ćemo ga od sljedećeg poglavlja.

Korak 4: Kreiranje Home/Index.cshtml

Lokacija datoteke: Views/Home/Index.cshtml
Desni klik na akciju Index() u HomeControlleru → Add View → ime: Index, Template: Empty, bez modela.

Alternativno: Kreirajte folder Views/Home/ i dodajte datoteku Index.cshtml. 
@{
    ViewBag.Title = "Početna";
}

<div class="jumbotron" style="padding: 2rem; background-color: #e9ecef; border-radius: .3rem; margin-bottom: 1rem;">
    <h1 class="display-4">AS.MVCDemo</h1>
    <p class="lead">ASP.NET MVC 5 demo aplikacija s EF6 DB First, CRUD operacijama i AJAX partial view-ovima.</p>
    <hr />
    <p>Koristite navigaciju za pristup modulima:</p>
    <a class="btn btn-primary btn-lg" href="@Url.Action("Index", "Department")">Odjeli</a>
    <a class="btn btn-secondary btn-lg" href="@Url.Action("Index", "Employee")" style="margin-left: 10px;">Zaposlenici</a>
</div>

Objašnjenje koda

ElementObjašnjenje
@{ ViewBag.Title = "Početna"; } Razor blok koji postavlja naslov. Ovaj naslov se koristi u Layout stranici: <title>@ViewBag.Title - AS.MVCDemo</title>
@Url.Action("Index", "Department") Generira URL string (npr. /Department) bez kreiranja kompletnog <a> taga. Koristimo ga unutar href atributa.
class="jumbotron" Bootstrap klasa za isticanje glavnog sadržaja stranice (hero section).
Url.Action vs Html.ActionLink:
  • @Url.Action("Index", "Department") — Vraća samo URL string: /Department
  • @Html.ActionLink("Odjeli", "Index", "Department") — Vraća kompletan HTML: <a href="/Department">Odjeli</a>
Koristite Url.Action kada želite ručno kontrolisati HTML tag (npr. unutar <a> ili <form>).

Rezultat

Nakon ova 4 koraka, pokrenite aplikaciju (F5 ili Ctrl+F5). Trebali biste vidjeti:

  • Tamnu navigacijsku traku na vrhu s linkovima na sve module
  • Početnu stranicu s naslovom "AS.MVCDemo" i dugmadima za Odjele i Zaposlenike
  • Footer s tekućom godinom
Linkovi u navigaciji još neće raditi (osim "Početna") jer još nismo kreirali ostale kontrolere. To ćemo uraditi u sljedećim poglavljima.

Šta ste naučili u ovom poglavlju

  • _Layout.cshtml je master template koji definira zajedničku strukturu svih stranica
  • @RenderBody() je placeholder gdje se ubacuje sadržaj svake stranice
  • @RenderSection omogućava stranicama da definiraju opcione sekcije (npr. za skripte)
  • _ViewStart.cshtml automatski postavlja Layout za sve View-ove
  • Kontroler mora naslijediti klasu Controller
  • ViewBag služi za prosljeđivanje malih podataka iz kontrolera u View
  • Html.ActionLink generira kompletne linkove, dok Url.Action generira samo URL
  • Bundles grupišu CSS i JS datoteke za bolje performanse
