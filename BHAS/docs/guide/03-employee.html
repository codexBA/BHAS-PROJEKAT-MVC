Poglavlje 3: EmployeeController

U ovom poglavlju kreirat ćemo kompletni CRUD za zaposlenike. Obrazac je isti kao za Department, ali s više polja u formi i dropdown listom za odabir odjela. Ovo poglavlje učvršćuje koncepte iz prethodnog poglavlja.

Sadržaj poglavlja: Controller | Index View | Details View | Create View | Edit View | Delete View

Korak 1: Kreiranje EmployeeController

Lokacija datoteke: Controllers/EmployeeController.cs
Desni klik na folder Controllers → Add → Controller → MVC 5 Controller - Empty → ime: EmployeeController 
using BHAS.DbFirst;
using System;
using System.Data.Entity;
using System.Linq;
using System.Web.Mvc;

namespace AS.MVCDemo.Controllers
{
    public class EmployeeController : Controller
    {
        // GET: Employee
        public ActionResult Index()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var employees = db.Employees
                    .OrderBy(x => x.LastName)
                    .ThenBy(x => x.FirstName)
                    .ToList();
                return View(employees);
            }
        }

        // GET: Employee/Details/5
        public ActionResult Details(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var employee = db.Employees
                    .Include("Department")
                    .FirstOrDefault(x => x.EmployeeID == id);

                if (employee == null)
                    return HttpNotFound();

                return View(employee);
            }
        }

        // GET: Employee/Create
        public ActionResult Create()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, null);
                return View();
            }
        }

        // POST: Employee/Create
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult Create([Bind(Include = "FirstName,LastName,JMBG,DepartmentID,Position,Salary,HireDate,Email")] Employee model)
        {
            if (ModelState.IsValid)
            {
                using (var db = new StateStatisticsDBEntities())
                {
                    model.CreatedDate = DateTime.Now;
                    model.ModifiedDate = DateTime.Now;
                    db.Employees.Add(model);
                    db.SaveChanges();
                    return RedirectToAction("Index");
                }
            }

            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, model.DepartmentID);
            }
            return View(model);
        }

        // GET: Employee/Edit/5
        public ActionResult Edit(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var employee = db.Employees.Find(id);
                if (employee == null)
                    return HttpNotFound();

                PopulateDepartmentsSelectList(db, employee.DepartmentID);
                return View(employee);
            }
        }

        // POST: Employee/Edit/5
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult Edit([Bind(Include = "EmployeeID,FirstName,LastName,JMBG,DepartmentID,Position,Salary,HireDate,Email")] Employee model)
        {
            if (ModelState.IsValid)
            {
                using (var db = new StateStatisticsDBEntities())
                {
                    model.ModifiedDate = DateTime.Now;
                    db.Entry(model).State = EntityState.Modified;
                    db.SaveChanges();
                    return RedirectToAction("Index");
                }
            }

            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, model.DepartmentID);
            }
            return View(model);
        }

        // GET: Employee/Delete/5
        public ActionResult Delete(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var employee = db.Employees
                    .Include("Department")
                    .FirstOrDefault(x => x.EmployeeID == id);

                if (employee == null)
                    return HttpNotFound();

                return View(employee);
            }
        }

        // POST: Employee/Delete/5
        [HttpPost, ActionName("Delete")]
        [ValidateAntiForgeryToken]
        public ActionResult DeleteConfirmed(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var employee = db.Employees.Find(id);
                if (employee == null)
                    return HttpNotFound();

                db.Employees.Remove(employee);
                db.SaveChanges();
                return RedirectToAction("Index");
            }
        }

        private void PopulateDepartmentsSelectList(StateStatisticsDBEntities db, int? selectedDepartmentId)
        {
            var departments = db.Departments
                .OrderBy(x => x.DepartmentName)
                .ToList();

            ViewBag.DepartmentID = new SelectList(departments, "DepartmentID", "DepartmentName", selectedDepartmentId);
        }
    }
}

Novi koncepti u ovom kontroleru

Include sa stringom vs lambda izraz:
U DepartmentController-u smo koristili: .Include(x => x.Employees) (lambda)
Ovdje koristimo: .Include("Department") (string)

Oba načina rade isto — učitavaju povezani entitet zajedno s glavnim. Lambda pristup daje IntelliSense podršku i provjeru u compile-time, dok string pristup ne daje.

Preporuka: Koristite lambda pristup kad god je moguće jer IDE javlja grešku ako pogrešno napišete ime.
PopulateDepartmentsSelectList:
Ova metoda je slična PopulateManagersSelectList iz DepartmentController-a, ali popunjava dropdown za odabir odjela umjesto menadžera.
ViewBag.DepartmentID = new SelectList(departments, "DepartmentID", "DepartmentName", selectedDepartmentId)
Parametri SelectList konstruktora: kolekcija, polje za value, polje za prikaz, trenutno odabrana vrijednost.

Formatiranje datuma:
@item.HireDate.ToString("dd.MM.yyyy") formatira datum kao npr. "15.03.2024".
Format tokeni: dd = dan (2 cifre), MM = mjesec (2 cifre), yyyy = godina (4 cifre).

Rad s nullable vrijednostima u Razor-u:
@(Model.Department != null ? Model.Department.DepartmentName : "-")
Ovo je ternarni operator: ako Department nije null, prikaži ime; inače prikaži "-".

@(string.IsNullOrEmpty(Model.Email) ? "-" : Model.Email)
Provjerava da li je string prazan ili null.

Novi koncept — type = "date" za datum:
@Html.EditorFor(m => m.HireDate, new { htmlAttributes = new { @class = "form-control", type = "date" } })

Dodavanjem type = "date" u htmlAttributes, browser prikazuje nativni date picker kontrolu umjesto običnog tekstualnog polja. Ovo značajno poboljšava korisničko iskustvo jer korisnik ne mora ručno upisivati format datuma.

Uočite razliku: Employee Delete nema try-catch blok u kontroleru, za razliku od Department-a. To je zato što zaposlenici obično nemaju zavisne zapise koji bi spriječili brisanje. Department ima zaposlenike koji zavise od njega, pa brisanje može baciti grešku.

Šta ste naučili u ovom poglavlju

  • Ponavljanje CRUD obrasca — Isti obrazac kao Department, ali s više polja
  • Include sa stringom — .Include("Department") kao alternativa lambda izrazu
  • Date picker — type = "date" za nativni browser date picker
  • Formatiranje datuma — ToString("dd.MM.yyyy")
  • Nullable provjere — HasValue, string.IsNullOrEmpty(), ternarni operator
  • Dropdown za FK — PopulateDepartmentsSelectList + DropDownList
