Poglavlje 8: Stored Procedures s Entity Framework 6

U ovom poglavlju naučit ćete kako pozivati pohranjene procedure (stored procedures) iz Entity Framework 6 na sva četiri načina — od čitanja postojećih entiteta, preko prilagođenih ViewModel oblika, do izvršavanja UPDATE naredbi i prikaza broja ažuriranih redova.

Novi koncepti u ovom poglavlju:
  • db.Database.SqlQuery<T>() — čitanje redova koji se mapiraju na postojeći EF entitet
  • db.Database.SqlQuery<ViewModel>() — čitanje prilagođenog oblika koji nije ni jedan EF entitet
  • db.Database.ExecuteSqlCommand() — izvršavanje UPDATE/DML procedure bez SELECT-a, vraća int
  • SqlParameter — siguran način prosljeđivanja parametara (zaštita od SQL injection)
  • Razlika između obaveznih i nullable parametara ((object)value ?? DBNull.Value)
  • ViewModeli čija svojstva moraju tačno odgovarati imenima kolona u proceduri
  • GET forme za filtriranje + POST forme za DML s inline prikazom rezultata

Pregled — 4 načina pozivanja stored procedure

Entity Framework 6 nudi dvije metode na db.Database objektu za rad s pohranjerim procedurama:

# Metoda Vraća Kada se koristi
1 SqlQuery<Employee>(...) Lista EF entitet objekata (praćenih) Procedura vraća sve kolone koje odgovaraju EF entitetu (npr. Employee)
2 SqlQuery<MojViewModel>(...) Lista obične C# klase (nije praćena) Procedura vraća JOIN/GROUP BY oblik koji ne odgovara ni jednom entitetu
3 ExecuteSqlCommand(...) → int Broj ažuriranih redova Procedura izvršava UPDATE bez SELECT — vraćeni int je broj ažuriranih redova
4 Isto kao 3 — prikazujemo int Broj ažuriranih redova (kao povratna informacija) Isto kao gore — int se prikazuje korisniku kao poruka o uspjehu
Kako EF mapira kolone na svojstva:
SqlQuery<T> mapira kolone iz rezultata procedure na svojstva C# klase prema imenu, bez obzira na velika/mala slova. Ako procedura vraća kolonu ManagerFullName, na klasi mora postojati svojstvo ManagerFullName (ili managerfullname — svejedno je). Ako kolona nema odgovarajuće svojstvo, ignorira se. Ako obavezno (non-nullable) svojstvo nema odgovarajuću kolonu, EF baci grešku pri izvršavanju.
SET NOCOUNT i ExecuteSqlCommand:
Metoda ExecuteSqlCommand vraća broj redova koji je ADO.NET primio od SQL Servera. Ako procedura poziva SET NOCOUNT ON, SQL Server ne šalje taj broj i ADO.NET vraća -1. Za sve procedure koje pozivate s ExecuteSqlCommand, uvijek koristite SET NOCOUNT OFF (ili ga uopće ne navođajte, jer je OFF zadana vrijednost).

Korak 1: SQL — 7 pohranjenih procedura

Lokacija datoteke: BHAS.DbFirst/SQL/stored-procedures.sql
Pokrenite ovaj skript u SSMS na bazi StateStatisticsDB prije testiranja aplikacije.

Sve procedure su u Stats shemi. Evo kratkog pregleda svake:

# Naziv Parametri Šta vraća EF6 metoda
1 sp_GetAllEmployees — Svi zaposlenici SqlQuery<Employee>
2 sp_GetEmployeesByDepartment @DepartmentID INT Zaposlenici odjela SqlQuery<Employee>
3 sp_SearchEmployees @DepartmentID, @MinSalary, @MaxSalary (svi nullable) Zaposlenici po filterima SqlQuery<Employee>
4 sp_GetDepartmentSummary — JOIN + GROUP BY agregati po odjelu SqlQuery<DepartmentSummaryViewModel>
5 sp_GetRegionalStatsByYear @Year INT Regionalna statistika + izračunati BDPpc SqlQuery<RegionalStatsViewModel>
6 sp_GiveSalaryRaise @DepartmentID, @PercentageIncrease UPDATE plate, bez SELECT ExecuteSqlCommand
7 sp_UpdateProjectStatus @ProjectID, @NewStatus UPDATE statusa projekta, bez SELECT ExecuteSqlCommand

Opcioni parametri u SQL-u — obrazac IS NULL OR

Procedura sp_SearchEmployees prima tri parametra koji su svi opcioni (zadana vrijednost je NULL). Kada parametar nije proslijeđen, taj filter se preskače:

CREATE PROCEDURE Stats.sp_SearchEmployees
    @DepartmentID  INT           = NULL,
    @MinSalary     DECIMAL(18,2) = NULL,
    @MaxSalary     DECIMAL(18,2) = NULL
AS
BEGIN
    SET NOCOUNT ON;
    SELECT ...
    FROM Stats.Employees
    WHERE
        (@DepartmentID IS NULL OR DepartmentID = @DepartmentID)
        AND (@MinSalary IS NULL OR Salary >= @MinSalary)
        AND (@MaxSalary IS NULL OR Salary <= @MaxSalary);
END
Obrazac (@Param IS NULL OR kolona = @Param):
Ovo znači: "ako pozivač proslijedi NULL, ignoriši ovaj filter; inače primijeni ga". Ovo je standardni SQL Server pristup za opcione WHERE uvjete unutar pohranjene procedure.

Izračunata kolona u SQL-u — BDP per capita

Procedura sp_GetRegionalStatsByYear vraća kolonu GDPPerCapita koja ne postoji ni u jednoj tabeli — izračunava se direktno u SQL-u:

CASE
    WHEN r.Population > 0 AND ed.GDP IS NOT NULL
    THEN ed.GDP / r.Population
    ELSE NULL
END AS GDPPerCapita

Provjera Population > 0 sprečava dijeljenje s nulom. Rezultat je nullable decimal u ViewModelu.

Zašto decimal? za ekonomske vrijednosti?
Procedura radi LEFT JOIN, pa regija može nemati podataka za traženu godinu. U tom slučaju SQL vraća NULL za sve numeričke kolone. Svojstvo tipa decimal? pravilno se mapira na SQL NULL. Obično decimal bi bacilo grešku pri konverziji kada EF pokuša popuniti to svojstvo iz NULL vrijednosti.

Kompletan kod kontrolera

using AS.MVCDemo.Models;
using BHAS.DbFirst;
using System.Data.SqlClient;
using System.Linq;
using System.Web.Mvc;

namespace AS.MVCDemo.Controllers
{
    public class StoredProcController : Controller
    {
        // GET: StoredProc
        public ActionResult Index()
        {
            return View();
        }

        // ── SP 1: SqlQuery<Employee> — bez parametara ──────────────────────────
        public ActionResult AllEmployees()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var list = db.Database
                             .SqlQuery<Employee>("EXEC Stats.sp_GetAllEmployees")
                             .ToList();
                return View(list);
            }
        }

        // ── SP 2: SqlQuery<Employee> — jedan parametar ─────────────────────────
        public ActionResult ByDepartment(int? departmentId)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, departmentId);

                if (!departmentId.HasValue)
                    return View((object)null);  // ← cast na object je obavezan (vidi objašnjenje)

                var p = new SqlParameter("@DepartmentID", departmentId.Value);
                var list = db.Database
                             .SqlQuery<Employee>(
                                 "EXEC Stats.sp_GetEmployeesByDepartment @DepartmentID",
                                 p)
                             .ToList();

                ViewBag.FilterDepartment = db.Departments.Find(departmentId.Value)?.DepartmentName;
                return View(list);
            }
        }

        // ── SP 3: SqlQuery<Employee> — nullable parametri ──────────────────────
        public ActionResult SearchEmployees(int? deptId, decimal? min, decimal? max)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, deptId);

                var p1 = new SqlParameter("@DepartmentID", (object)deptId ?? System.DBNull.Value);
                var p2 = new SqlParameter("@MinSalary",    (object)min    ?? System.DBNull.Value);
                var p3 = new SqlParameter("@MaxSalary",    (object)max    ?? System.DBNull.Value);

                var list = db.Database
                             .SqlQuery<Employee>(
                                 "EXEC Stats.sp_SearchEmployees @DepartmentID, @MinSalary, @MaxSalary",
                                 p1, p2, p3)
                             .ToList();

                return View(list);
            }
        }

        // ── SP 4: SqlQuery<ViewModel> — prilagođeni oblik, bez parametara ─────
        public ActionResult DepartmentSummary()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var list = db.Database
                             .SqlQuery<DepartmentSummaryViewModel>(
                                 "EXEC Stats.sp_GetDepartmentSummary")
                             .ToList();
                return View(list);
            }
        }

        // ── SP 5: SqlQuery<ViewModel> — prilagođeni oblik, jedan parametar ────
        public ActionResult RegionalStats(int? year)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var years = db.EconomicDatas
                              .Select(x => x.Year)
                              .Distinct()
                              .OrderByDescending(y => y)
                              .ToList();
                ViewBag.Years = new SelectList(years);

                if (!year.HasValue)
                    return View((object)null);

                ViewBag.SelectedYear = year.Value;
                var p = new SqlParameter("@Year", year.Value);
                var list = db.Database
                             .SqlQuery<RegionalStatsViewModel>(
                                 "EXEC Stats.sp_GetRegionalStatsByYear @Year",
                                 p)
                             .ToList();
                return View(list);
            }
        }

        // ── SP 6: ExecuteSqlCommand — GET forma ────────────────────────────────
        public ActionResult SalaryRaise()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, null);
                return View();
            }
        }

        // ── SP 6: ExecuteSqlCommand — POST ─────────────────────────────────────
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult SalaryRaise(int departmentId, decimal percentage)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var p1 = new SqlParameter("@DepartmentID",       departmentId);
                var p2 = new SqlParameter("@PercentageIncrease", percentage);

                int rows = db.Database.ExecuteSqlCommand(
                    "EXEC Stats.sp_GiveSalaryRaise @DepartmentID, @PercentageIncrease",
                    p1, p2);

                ViewBag.RowsAffected = rows;         // ← šalje se view-u za prikaz poruke
                PopulateDepartmentsSelectList(db, departmentId);
                return View();                       // ← ostaje na istoj stranici (ne redirect)
            }
        }

        // ── SP 7: ExecuteSqlCommand — GET forma ────────────────────────────────
        public ActionResult UpdateProjectStatus()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                PopulateProjectsSelectList(db, null);
                PopulateStatusSelectList(null);
                return View();
            }
        }

        // ── SP 7: ExecuteSqlCommand — POST ─────────────────────────────────────
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult UpdateProjectStatus(int projectId, string newStatus)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var p1 = new SqlParameter("@ProjectID", projectId);
                var p2 = new SqlParameter("@NewStatus", newStatus);

                int rows = db.Database.ExecuteSqlCommand(
                    "EXEC Stats.sp_UpdateProjectStatus @ProjectID, @NewStatus",
                    p1, p2);

                ViewBag.RowsAffected = rows;
                PopulateProjectsSelectList(db, projectId);
                PopulateStatusSelectList(newStatus);
                return View();
            }
        }

        // ── Pomoćne metode ──────────────────────────────────────────────────────
        private void PopulateDepartmentsSelectList(StateStatisticsDBEntities db, int? selectedId)
        {
            var departments = db.Departments.OrderBy(x => x.DepartmentName).ToList();
            ViewBag.DepartmentID = new SelectList(departments, "DepartmentID", "DepartmentName", selectedId);
        }

        private void PopulateProjectsSelectList(StateStatisticsDBEntities db, int? selectedId)
        {
            var projects = db.Projects.OrderBy(x => x.ProjectName).ToList();
            ViewBag.ProjectID = new SelectList(projects, "ProjectID", "ProjectName", selectedId);
        }

        private void PopulateStatusSelectList(string selectedStatus)
        {
            var statuses = new[]
            {
                new { Value = "Planned",    Text = "Planiran" },
                new { Value = "InProgress", Text = "U toku"   },
                new { Value = "Completed",  Text = "Završen"  },
                new { Value = "Cancelled",  Text = "Otkazan"  }
            };
            ViewBag.StatusList = new SelectList(statuses, "Value", "Text", selectedStatus);
        }
    }
}

Detaljno objašnjenje ključnih dijelova

1. SqlQuery<T> — kako pozvati proceduru

Bez parametara:
db.Database.SqlQuery<Employee>("EXEC Stats.sp_GetAllEmployees")

S jednim parametrom:
var p = new SqlParameter("@DepartmentID", departmentId.Value);
db.Database.SqlQuery<Employee>("EXEC Stats.sp_GetAllEmployees @DepartmentID", p)

String koji prosljeđujete je isti SQL koji biste upisali u SSMS. Uvijek koristite SqlParameter — nikad string interpolaciju ili konkatenaciju, jer je to SQL injection ranjivost.

2. return View((object)null) — zašto je cast obavezan

Kada akcija nema podataka za prikaz (npr. nije odabran odjel), šaljemo null model view-u. Ali return View(null) je dvosmisleno — kompajler ne zna radi li se o View(string viewName) ili View(object model) overloadu. Cast (object)null razrješava tu dvosmislenost. U view-u zatim provjerite: @if (Model != null) { ... }

3. ExecuteSqlCommand i return View() umjesto redirect-a

Za DML akcije (SalaryRaise, UpdateProjectStatus) ostajemo na istoj stranici umjesto da preusmjeravamo na Index. Razlog: trebamo prikazati poruku korisniku ("3 zaposlenika ažurirano"). Redirect bi obrisao ViewBag.

Pravilo: return View() koristite samo za ovakve feedback akcije, ne za Create/Edit gdje bi refresh stranice ponovio POST zahtjev.

4. PopulateStatusSelectList — statički dropdown bez baze

Status projekta (Planned, InProgress, Completed, Cancelled) su fiksne vrijednosti definirane CHECK constraintom u bazi. Ne trebamo upit na bazu — koristimo anonimni niz s Value i Text svojstvima. SelectList prima ta svojstva po imenu kao string parametre.

Korak 4: SqlParameter — obavezni vs nullable parametri

Ovo je najčešće mjesto greške pri pozivanju procedura. Postoje dva slučaja:

Obavezni parametar (ne može biti null)

// Jednostavno: prosljeđujemo vrijednost direktno
var p = new SqlParameter("@DepartmentID", departmentId);

Nullable / opcioni parametar

// C# null se NE može direktno proslijediti — ADO.NET baci ArgumentNullException
// Morate pretvoriti null → DBNull.Value

var p = new SqlParameter("@DepartmentID", (object)departmentId ?? DBNull.Value);
//                                         ↑ cast na object je neophodan
Zašto (object) cast?
Bez casta na (object), operator ?? radi na tipu int?, a desna strana DBNull.Value je drugačiji tip — izraz se ne kompajlira. Cast na (object) pakuje nullable integer kao object, čineći obje strane ?? istog tipa. Kada je vrijednost null, izraz postaje DBNull.Value, što ADO.NET ispravno prevodi u SQL NULL.

Ovaj obrazac radi za sve nullable tipove: int?, decimal?, DateTime? itd.

Primjer s tri nullable parametra

var p1 = new SqlParameter("@DepartmentID", (object)deptId ?? System.DBNull.Value);
var p2 = new SqlParameter("@MinSalary",    (object)min    ?? System.DBNull.Value);
var p3 = new SqlParameter("@MaxSalary",    (object)max    ?? System.DBNull.Value);

var list = db.Database
             .SqlQuery<Employee>(
                 "EXEC Stats.sp_SearchEmployees @DepartmentID, @MinSalary, @MaxSalary",
                 p1, p2, p3)
             .ToList();
Redoslijed parametara u EXEC stringu mora odgovarati SqlParameter objektima!
Ime placeholder-a u SQL stringu (@DepartmentID) mora se podudarati s imenom koje ste dali SqlParameter objektu ("@DepartmentID"). Redoslijed u EXEC stringu i redoslijed objekata koji se prosljeđuju metodi moraju biti konzistentni.

Atribut [ValidateAntiForgeryToken] na POST akciji odbija svaki zahtjev koji ne sadrži token koji generiše @Html.AntiForgeryToken(). Ako zaboravite token u view-u, POST će vratiti 400 grešku. Ovo je standardna CSRF zaštita u ASP.NET MVC.

Korak 6: Demo klasa za konzolnu aplikaciju

Lokacija datoteke: BHAS.DbFirst/Demo/StoredProcExamples.cs

Ova klasa prati isti obrazac kao ReadingExamples.cs — svaka metoda je samostalna s vlastitim using (var db = ...) blokom. Poziva se iz konzolne aplikacije (AS.ConsoleDbDemo/Program.cs).

using System.Collections.Generic;
using System.Data.SqlClient;  // ← OBAVEZNO za SqlParameter
using System.Linq;

namespace BHAS.DbFirst.Demo
{
    public class StoredProcExamples
    {
        // SP 1 — SqlQuery<Employee>, bez parametara
        public List<Employee> GetAllEmployees()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                return db.Database
                         .SqlQuery<Employee>("EXEC Stats.sp_GetAllEmployees")
                         .ToList();
            }
        }

        // SP 3 — SqlQuery<Employee>, nullable parametri
        public List<Employee> SearchEmployees(
            int? departmentId = null, decimal? minSalary = null, decimal? maxSalary = null)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var p1 = new SqlParameter("@DepartmentID", (object)departmentId ?? DBNull.Value);
                var p2 = new SqlParameter("@MinSalary",    (object)minSalary    ?? DBNull.Value);
                var p3 = new SqlParameter("@MaxSalary",    (object)maxSalary    ?? DBNull.Value);

                return db.Database
                         .SqlQuery<Employee>(
                             "EXEC Stats.sp_SearchEmployees @DepartmentID, @MinSalary, @MaxSalary",
                             p1, p2, p3)
                         .ToList();
            }
        }

        // SP 4 — SqlQuery<DepartmentSummaryViewModel>, prilagođeni oblik
        public List<DepartmentSummaryViewModel> GetDepartmentSummary()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                return db.Database
                         .SqlQuery<DepartmentSummaryViewModel>(
                             "EXEC Stats.sp_GetDepartmentSummary")
                         .ToList();
            }
        }

        // SP 6 — ExecuteSqlCommand, vraća broj ažuriranih redova
        public int GiveSalaryRaise(int departmentId, decimal percentageIncrease)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var p1 = new SqlParameter("@DepartmentID",       departmentId);
                var p2 = new SqlParameter("@PercentageIncrease", percentageIncrease);

                return db.Database.ExecuteSqlCommand(
                    "EXEC Stats.sp_GiveSalaryRaise @DepartmentID, @PercentageIncrease",
                    p1, p2);
            }
        }
    }
}

        new { area = "" }, new { @class = "nav-link" })</li>
<!-- Dodajte ovaj red: -->
<li>@Html.ActionLink("Stored Proc", "Index", "StoredProc",
        new { area = "" }, new { @class = "nav-link" })</li>

Šta ste naučili u ovom poglavlju

  • 4 EF6 obrasca za pozivanje procedura — SqlQuery<Entity>, SqlQuery<ViewModel>, ExecuteSqlCommand (s i bez prikaza rezultata)
  • SqlParameter — jedini siguran način prosljeđivanja parametara, zaštita od SQL injection napada
  • Nullable parametri — (object)value ?? DBNull.Value obrazac i zašto je (object) cast obavezan
  • ViewModeli — obične C# klase čija imena svojstava moraju tačno odgovarati imenima kolona u proceduri
  • SET NOCOUNT OFF — obavezno za procedure koje se pozivaju s ExecuteSqlCommand
  • return View((object)null) — kako poslati null model view-u bez kompajlerske dvosmislenosti
  • return View() umjesto redirect-a — kada je ispravno ostati na istoj stranici i prikazati feedback
  • Izračunate kolone u SQL-u — mogu se mapirati na ViewModel svojstva kao da su normalne kolone
