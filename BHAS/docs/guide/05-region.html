Poglavlje 5: RegionController

U ovom poglavlju kreirat ćemo CRUD za regije, uključujući AJAX partial view za ekonomske podatke i cross-controller linkove. Region je sličan Department-u jer koristi partial view i AJAX, ali uvodi linkove prema drugom kontroleru (EconomicData).

Novi koncepti u ovom poglavlju:
  • Cross-controller ActionLink (linkovi na akcije u drugom kontroleru)
  • Partial view s akcijskim linkovima na drugi kontroler
  • CRUD bez dropdown-a (Region nema FK na drugi entitet)

Korak 1: Kreiranje RegionController

Lokacija datoteke: Controllers/RegionController.cs 
using BHAS.DbFirst;
using System;
using System.Data.Entity;
using System.Linq;
using System.Web.Mvc;

namespace AS.MVCDemo.Controllers
{
    public class RegionController : Controller
    {
        // GET: Region
        public ActionResult Index()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var regions = db.Regions
                    .OrderBy(x => x.RegionName)
                    .ToList();
                return View(regions);
            }
        }

        // GET: Region/Details/5
        public ActionResult Details(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var region = db.Regions
                    .Include(x => x.EconomicDatas)
                    .Include(x => x.ReportDatas)
                    .FirstOrDefault(x => x.RegionID == id);

                if (region == null)
                    return HttpNotFound();

                return View(region);
            }
        }

        // GET: Region/Create
        public ActionResult Create()
        {
            return View();
        }

        // POST: Region/Create
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult Create([Bind(Include = "RegionName,RegionCode,Population,Area")] Region model)
        {
            if (ModelState.IsValid)
            {
                using (var db = new StateStatisticsDBEntities())
                {
                    model.CreatedDate = DateTime.Now;
                    model.ModifiedDate = DateTime.Now;
                    db.Regions.Add(model);
                    db.SaveChanges();
                    return RedirectToAction("Index");
                }
            }
            return View(model);
        }

        // GET: Region/Edit/5
        public ActionResult Edit(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var region = db.Regions.Find(id);
                if (region == null)
                    return HttpNotFound();
                return View(region);
            }
        }

        // POST: Region/Edit/5
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult Edit([Bind(Include = "RegionID,RegionName,RegionCode,Population,Area")] Region model)
        {
            if (ModelState.IsValid)
            {
                using (var db = new StateStatisticsDBEntities())
                {
                    model.ModifiedDate = DateTime.Now;
                    db.Entry(model).State = EntityState.Modified;
                    db.SaveChanges();
                    return RedirectToAction("Index");
                }
            }
            return View(model);
        }

        // GET: Region/Delete/5
        public ActionResult Delete(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var region = db.Regions.Find(id);
                if (region == null)
                    return HttpNotFound();
                return View(region);
            }
        }

        // POST: Region/Delete/5
        [HttpPost, ActionName("Delete")]
        [ValidateAntiForgeryToken]
        public ActionResult DeleteConfirmed(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                try
                {
                    var region = db.Regions.Find(id);
                    if (region == null)
                        return HttpNotFound();

                    db.Regions.Remove(region);
                    db.SaveChanges();
                    return RedirectToAction("Index");
                }
                catch (Exception)
                {
                    TempData["Error"] = "Nije moguće obrisati regiju jer ima vezanih podataka (ekonomski podaci ili podaci izvještaja).";
                    return RedirectToAction("Details", new { id });
                }
            }
        }

        // GET: Region/EconomicDataPartial?regionId=5
        [HttpGet]
        public ActionResult EconomicDataPartial(int regionId)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var data = db.EconomicDatas
                    .Where(x => x.RegionID == regionId)
                    .OrderByDescending(x => x.Year)
                    .ToList();
                return PartialView("_EconomicDataTable", data);
            }
        }
    }
}
Razlike od DepartmentController-a:
  • Create GET je najjednostavniji oblik — samo return View(), bez dropdown-a jer Region nema FK polja
  • Create/Edit POST — Ako validacija ne uspije, samo vraćamo View(model) bez ponovnog popunjavanja dropdown-a
  • EconomicDataPartial — Ista logika kao EmployeesPartial, ali dohvaća ekonomske podatke za regiju
  • Delete — Ima try-catch jer regija može imati vezane ekonomske podatke

@section scripts {
    <script>
        $(function () {
            $("#btnLoadEcoData").on("click", function () {
                var regionId = $(this).data("region-id");
                var $host = $("#ecoDataHost");

                $host.html("<div class='text-muted'>Učitavam...</div>");

                $.get("/Region/EconomicDataPartial", { regionId: regionId })
                    .done(function (html) { $host.html(html); })
                    .fail(function (xhr) {
                        $host.html("<div class='alert alert-danger'>Greška: " + xhr.status + "</div>");
                    });
            });
        });
    </script>
}
Cross-controller ActionLink:
@Html.ActionLink("Dodaj ekonomske podatke", "Create", "EconomicData", new { regionId = Model.RegionID }, new { @class = "btn btn-success" })

Parametri su:
  1. "Dodaj ekonomske podatke" — Tekst linka
  2. "Create" — Akcija u drugom kontroleru
  3. "EconomicData" — Ime ciljnog kontrolera
  4. new { regionId = Model.RegionID } — Query string parametar (generira URL: /EconomicData/Create?regionId=5)
  5. new { @class = "btn btn-success" } — HTML atributi

Ovo je razlika od @Html.ActionLink("Uredi", "Edit", new { id = ... }) koji cilja akciju u istom kontroleru.

Cross-controller linkovi u Partial View-u:
@Html.ActionLink("Uredi", "Edit", "EconomicData", new { id = item.DataID }, ...)

Ovaj partial view pripada Region kontroleru, ali akcijski linkovi vode na EconomicData kontroler. Ovo demonstrira kako partial view-ovi mogu sadržavati linkove na bilo koji kontroler u aplikaciji.

Generira URL: /EconomicData/Edit/5

Šta ste naučili u ovom poglavlju

  • Jednostavan Create — Bez dropdown-a kada entitet nema FK
  • Cross-controller ActionLink — Linkovi na akcije u drugom kontroleru s query string parametrima
  • Partial view s cross-controller linkovima — _EconomicDataTable koristi EconomicData kontroler
  • AJAX pattern (ponavljanje) — Isti $.get() obrazac kao kod Department-a
  • Try-catch pri brisanju — Zaštita od FK constraint grešaka
