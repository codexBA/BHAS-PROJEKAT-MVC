Poglavlje 7: ReportController

U posljednjem poglavlju kreirat ćemo kontroler za izvještaje. Ovaj kontroler je poseban jer: nema Delete akciju, koristi višestruke dropdown-e (odjel, zaposlenik, status, tip), i prikazuje child kolekciju (ReportDatas) u Details View-u.

Novi koncepti u ovom poglavlju:
  • CRUD bez Delete akcije (svjesna dizajn odluka)
  • Višestruke PopulateSelectList metode za različite dropdown-e
  • Kombinacija statičkih i dinamičkih dropdown-a
  • Nested Include za povezane entitete (ReportDatas → Region)
  • Prikaz child kolekcije u Details View-u
  • RedirectToAction na Details nakon Create

Korak 1: Kreiranje ReportController

Lokacija datoteke: Controllers/ReportController.cs 
using BHAS.DbFirst;
using System;
using System.Data.Entity;
using System.Linq;
using System.Web.Mvc;

namespace AS.MVCDemo.Controllers
{
    public class ReportController : Controller
    {
        private static readonly string[] StatusValues = { "Draft", "U obradi", "Završen", "Arhiviran" };
        private static readonly string[] ReportTypeValues = { "Statistički", "Finansijski", "Analitički", "Operativni" };

        // GET: Report
        public ActionResult Index()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var reports = db.Reports
                    .Include(x => x.Department)
                    .Include(x => x.Employee)
                    .OrderByDescending(x => x.CreatedDate)
                    .ToList();
                return View(reports);
            }
        }

        // GET: Report/Details/5
        public ActionResult Details(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var report = db.Reports
                    .Include(x => x.Department)
                    .Include(x => x.Employee)
                    .Include(x => x.ReportDatas.Select(rd => rd.Region))
                    .FirstOrDefault(x => x.ReportID == id);

                if (report == null)
                    return HttpNotFound();

                return View(report);
            }
        }

        // GET: Report/Create
        public ActionResult Create()
        {
            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, null);
                PopulateEmployeesSelectList(db, null);
                ViewBag.StatusList = new SelectList(StatusValues);
                ViewBag.ReportTypeList = new SelectList(ReportTypeValues);
                return View();
            }
        }

        // POST: Report/Create
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult Create([Bind(Include = "ReportName,DepartmentID,CreatedBy,Status,ReportType,Description")] Report model)
        {
            if (ModelState.IsValid)
            {
                using (var db = new StateStatisticsDBEntities())
                {
                    model.CreatedDate = DateTime.Now;
                    db.Reports.Add(model);
                    db.SaveChanges();
                    return RedirectToAction("Details", new { id = model.ReportID });
                }
            }

            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, model.DepartmentID);
                PopulateEmployeesSelectList(db, model.CreatedBy);
                ViewBag.StatusList = new SelectList(StatusValues, model.Status);
                ViewBag.ReportTypeList = new SelectList(ReportTypeValues, model.ReportType);
            }
            return View(model);
        }

        // GET: Report/Edit/5
        public ActionResult Edit(int id)
        {
            using (var db = new StateStatisticsDBEntities())
            {
                var report = db.Reports.Find(id);
                if (report == null)
                    return HttpNotFound();

                PopulateDepartmentsSelectList(db, report.DepartmentID);
                PopulateEmployeesSelectList(db, report.CreatedBy);
                ViewBag.StatusList = new SelectList(StatusValues, report.Status);
                ViewBag.ReportTypeList = new SelectList(ReportTypeValues, report.ReportType);
                return View(report);
            }
        }

        // POST: Report/Edit/5
        [HttpPost]
        [ValidateAntiForgeryToken]
        public ActionResult Edit([Bind(Include = "ReportID,ReportName,DepartmentID,CreatedBy,Status,ReportType,Description")] Report model)
        {
            if (ModelState.IsValid)
            {
                using (var db = new StateStatisticsDBEntities())
                {
                    db.Entry(model).State = EntityState.Modified;
                    db.SaveChanges();
                    return RedirectToAction("Details", new { id = model.ReportID });
                }
            }

            using (var db = new StateStatisticsDBEntities())
            {
                PopulateDepartmentsSelectList(db, model.DepartmentID);
                PopulateEmployeesSelectList(db, model.CreatedBy);
                ViewBag.StatusList = new SelectList(StatusValues, model.Status);
                ViewBag.ReportTypeList = new SelectList(ReportTypeValues, model.ReportType);
            }
            return View(model);
        }

        private void PopulateDepartmentsSelectList(StateStatisticsDBEntities db, int? selectedId)
        {
            var departments = db.Departments.OrderBy(x => x.DepartmentName).ToList();
            ViewBag.DepartmentID = new SelectList(departments, "DepartmentID", "DepartmentName", selectedId);
        }

        private void PopulateEmployeesSelectList(StateStatisticsDBEntities db, int? selectedId)
        {
            var employees = db.Employees
                .OrderBy(x => x.LastName).ThenBy(x => x.FirstName)
                .Select(x => new { x.EmployeeID, FullName = x.LastName + " " + x.FirstName })
                .ToList();
            ViewBag.CreatedBy = new SelectList(employees, "EmployeeID", "FullName", selectedId);
        }
    }
}

Detaljno objašnjenje novih koncepata

1. Bez Delete akcije

ReportController nema Delete i DeleteConfirmed akcije. Ovo je svjesna dizajn odluka — izvještaji su važni dokumenti i ne bi trebali biti brisani. U stvarnom projektu, umjesto brisanja biste mogli implementirati "arhiviranje" (promjena statusa na "Arhiviran").

Ovo pokazuje da ne morate uvijek implementirati svih 5 CRUD akcija. Implementirajte samo ono što ima smisla za vaš poslovni slučaj.

2. Višestruke PopulateSelectList metode

Ovaj kontroler ima dvije privatne metode za popunjavanje dropdown-a:
  • PopulateDepartmentsSelectList — Za odabir odjela (ViewBag.DepartmentID)
  • PopulateEmployeesSelectList — Za odabir kreatora (ViewBag.CreatedBy)
Plus dva statička dropdown-a:
  • ViewBag.StatusList — Status izvještaja (Draft, U obradi, Završen, Arhiviran)
  • ViewBag.ReportTypeList — Tip izvještaja (Statistički, Finansijski, Analitički, Operativni)
Svaki od ova 4 dropdown-a mora biti popunjen u svim akcijama koje prikazuju formu (Create GET, Create POST s greškom, Edit GET, Edit POST s greškom).

3. Nested Include

.Include(x => x.ReportDatas.Select(rd => rd.Region))

Ovo je nested eager loading. Učitava ne samo ReportDatas kolekciju, nego i Region objekat za svaki ReportData.

Bez ovoga, u Details View-u bismo mogli pristupiti rd.Region ali bi bio null (jer nije učitan iz baze).

4. Redirect na Details nakon Create

return RedirectToAction("Details", new { id = model.ReportID });

Za razliku od ostalih kontrolera koji preusmjeravaju na Index, ovaj preusmjerava na Details. Ovo je korisno jer korisnik odmah vidi kreirani izvještaj s svim detaljima.

model.ReportID je dostupan nakon db.SaveChanges() jer EF automatski popunjava ID nakon insert-a.

Korak 2: Index View (Lista izvještaja)

Lokacija datoteke: Views/Report/Index.cshtml
Kreirajte folder Views/Report/, zatim dodajte Index.cshtml. 
@model List<BHAS.DbFirst.Report>
@{
    ViewBag.Title = "Izvještaji";
}

<h2>Izvještaji</h2>
<p>@Html.ActionLink("Kreiraj novi izvještaj", "Create", null, new { @class = "btn btn-primary" })</p>

<table class="table table-bordered table-striped table-hover">
    <thead class="table-dark">
        <tr>
            <th>Naziv izvještaja</th>
            <th>Odjel</th>
            <th>Kreirao</th>
            <th>Tip</th>
            <th>Status</th>
            <th>Datum</th>
            <th>Akcije</th>
        </tr>
    </thead>
    <tbody>
        @foreach (var item in Model)
        {
            <tr>
                <td>@item.ReportName</td>
                <td>@(item.Department != null ? item.Department.DepartmentName : "-")</td>
                <td>@(item.Employee != null ? item.Employee.LastName + " " + item.Employee.FirstName : "-")</td>
                <td>@item.ReportType</td>
                <td>
                    @{
                        var cls = item.Status == "Završen" ? "bg-success"
                            : item.Status == "Arhiviran" ? "bg-secondary"
                            : item.Status == "U obradi" ? "bg-warning text-dark"
                            : "bg-info text-dark";
                    }
                    <span class="badge @cls">@item.Status</span>
                </td>
                <td>@(item.CreatedDate.HasValue ? item.CreatedDate.Value.ToString("dd.MM.yyyy") : "-")</td>
                <td>
                    @Html.ActionLink("Detalji", "Details", new { id = item.ReportID }, new { @class = "btn btn-sm btn-info" })
                    @Html.ActionLink("Uredi", "Edit", new { id = item.ReportID }, new { @class = "btn btn-sm btn-warning" })
                </td>
            </tr>
        }
    </tbody>
</table>
Uočite: U koloni Akcije nema linka "Obriši". Ovo odgovara dizajn odluci da izvještaji ne mogu biti brisani. Također, prikaz navigacijskog svojstva Employee koristi format: item.Employee.LastName + " " + item.Employee.FirstName.

                    <td>@(rd.Region != null ? rd.Region.RegionName : "-")</td>

}
Prikaz child kolekcije:
Ovo je ključni novi koncept — unutar Details View-a prikazujemo i listu povezanih zapisa (ReportDatas).

@Model.ReportDatas.Count — Prikazuje broj zapisa u zaglavlju.
@foreach (var rd in Model.ReportDatas.OrderBy(x => x.Year).ThenBy(x => x.DataType)) — Iterira kroz podatke izvještaja, sortirane po godini i tipu.

Ovo je moguće jer smo u kontroleru koristili: .Include(x => x.ReportDatas.Select(rd => rd.Region))
Bez tog Include-a, Model.ReportDatas bi bila prazna kolekcija.

4 dropdown-a u jednoj formi:
Ova forma demonstrira kombinaciju:
  • 2 dinamička dropdown-a iz baze: DepartmentID i CreatedBy koriste null kao drugi parametar jer se ViewBag naziv poklapa s imenom DropDownList-a
  • 2 statička dropdown-a: ReportType i Status zahtijevaju eksplicitni cast (SelectList)ViewBag.ReportTypeList jer se ViewBag naziv razlikuje

Šta ste naučili u ovom poglavlju

  • Parcijalni CRUD — Ne morate implementirati sve CRUD operacije; samo ono što poslovni zahtjevi traže
  • Višestruki dropdown-ovi — Kombinacija dinamičkih (iz baze) i statičkih (niz stringova)
  • Višestruke Populate metode — Svaka popunjava svoj ViewBag za odgovarajući dropdown
  • Nested Include — .Include(x => x.ReportDatas.Select(rd => rd.Region))
  • Child kolekcija u View-u — Prikaz povezanih zapisa (ReportDatas) unutar Details stranice
  • Redirect na Details — Umjesto na Index, preusmjeravanje na Details nakon Create za bolji UX

Čestitamo! Završili ste vodič.

Sada znate kako:

  • Kreirati ASP.NET MVC kontrolere s CRUD operacijama
  • Koristiti Entity Framework za pristup bazi podataka
  • Kreirati Razor View-ove s formama, tabelama i karticama
  • Koristiti HTML helpere (EditorFor, LabelFor, DropDownList, TextAreaFor, itd.)
  • Implementirati dropdown liste (dinamičke iz baze i statičke)
  • Kreirati partial view-ove za ponovnu upotrebu i AJAX pozive
  • Koristiti jQuery za asihrone AJAX zahtjeve
  • Raditi s navigacijskim svojstvima i eager loading-om
  • Upravljati navigacijom između kontrolera (cross-controller linkovi i redirect-i)
  • Filtrirati podatke s GET formama i opcionim parametrima
  • Koristiti TempData za poruke kroz preusmjeravanje
  • Zaštititi forme od CSRF napada s AntiForgeryToken
